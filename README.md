
# CPU Affinity Tool

Простая утилита для управления привязкой процессов к ядрам процессора (CPU Affinity) с дашбордом.

**Учебный проект** 

## Возможности

- **Веб-дашборд** с реал-тайм обновлением списка процессов
- Просмотр процессов с сортировкой по загрузке CPU
- Отображение текущей привязки (affinity) к ядрам
- Установка привязки процессов к конкретным ядрам через чекбоксы
- Функция «закрепления» (заморозки) важных процессов в верхней части списка
- Поддержка Windows и Linux
- Автоматическое открытие браузера при запуске
- Работает как standalone EXE (через PyInstaller)

## Технологии

- **Python** + **FastAPI**
- **psutil** для работы с процессами
- Tailwind CSS + vanilla JS для фронтенда
- Uvicorn как ASGI-сервер

## Установка

### Вариант 1: Через Python

```bash
# Клонируйте репозиторий
git clone https://github.com/aDlnOcOM/cpu-affinity-tool.git
cd cpu-affinity-tool

# Установите зависимости
pip install -r requirements.txt
```

### Вариант 2: Готовый EXE (рекомендуется)

Скачайте последнюю версию из [Releases](https://github.com/aDlnOcOM/cpu-affinity-tool/releases).  
Запускайте **от имени администратора** (Windows) или root (Linux) для возможности назначения ядер на процессы.

## Использование

1. Запустите приложение:
   ```bash
   python main.py
   ```
   Или запустите скомпилированный `cpu-affinity-tool.exe`.

2. Автоматически откроется браузер по адресу `http://127.0.0.1:8000`.

3. В интерфейсе:
   - **CPU %** — процент загрузки (нормализован по количеству ядер)
   - Чекбоксы ядер — кликайте для изменения привязки процесса
   - Кнопка **📌 Закрепить** / **❄️ Открепить** — фиксирует процесс вверху списка


## Сборка EXE

```bat
# Windows
build.bat
```

Собранный файл появится в папке `dist/`.

## Структура проекта

- `main.py` — основной файл (FastAPI + веб-интерфейс)
- `requirements.txt` — зависимости
- `build.bat` — скрипт сборки EXE
- `hooks/` — git-хуки (если используются)

## Примечания

- Процессы, у которых affinity = `None`, отображаются как привязанные ко всем ядрам.
- Замороженные процессы сохраняются в localStorage браузера.
- Обновление списка процессов происходит каждые 3 секунды.
- Минимальное количество ядер для привязки — 1.
